## 0. このドキュメントの目的

第二フェーズで確認した課題をもとに、次に検証すべき **製品コンセプト** を定義する。

ここでやること:

- 何を「製品コンセプト」として見せるか決める
- 誰に見せるか決める
- 何を聞けば次の意思決定に進めるか決める
- 実装優先度を「学習したいこと」ベースで整理する

ここでやらないこと:

- すべての要望を実装リスト化しない
- 価格やLPを確定しない
- 駐在/ビジネス/ポーカー全方位に同時最適化しない

## 1. 検証する製品コンセプト

### 暫定コンセプト

> 現場で使った中国語を、次に使える形で残す会話・フレーズツール。

### もう少し具体化した説明

> 翻訳アプリのようにその場で中国語を理解・生成できる。  
> ただし、調べて終わりではなく、使った表現が自分専用ライブラリに残り、状況別に整理でき、覚えたいものだけドリル化できる。

### 既存ツールとの差分

| 既存ツール | できること | 足りないこと | 当方の差分 |
|---|---|---|---|
| Google翻訳 / DeepL | 即時翻訳 | 調べて終わる、現場文脈が残らない | 保存・整理・ドリル化 |
| ChatGPT | 自然な表現生成 | 会話中に使いづらい、履歴が学習に接続しにくい | 現場利用前提のライブラリ |
| Duolingo | 継続学習 | 自分の現場表現ではない | ユーザー固有フレーズ |
| Anki/メモ | 保存・暗記 | 作成が面倒、翻訳/音声/文脈がない | 翻訳から自動で学習素材化 |
| 語学レッスン | 対人・予定化 | 高コスト、現場の瞬間には間に合わない | 現場ログを次回練習に変換 |

## 2. 今検証したい中心仮説

### 仮説A: 会話補助が入口になる

ユーザーは「勉強したい」より先に、現場で以下をしたい。

- 相手の中国語を理解したい
- 自分の言いたいことを中国語にしたい
- 会話のテンポを止めすぎたくない
- 英語/笑顔/雰囲気で流す回数を減らしたい

検証したい問い:

```
日→中だけでなく中→日が入ると、会話中にこのアプリを開く理由が増えるか？
```

### 仮説B: 保存・整理がGoogle翻訳との差別化になる

翻訳アプリはその場で便利だが、後から見返しにくい。

検証したい問い:

```
翻訳した/困った表現が全件ライブラリに残り、状況別に整理できることは、Google翻訳よりこのアプリを使う理由になるか？
```

### 仮説C: ドリル化は「全部」ではなく「選択式」がよい

学習意欲が高い人にはドリルが刺さる。  
一方で、低学習意欲の人には全部ドリル化は重い。

検証したい問い:

```
覚えたいものだけドリル化できる設計は、未学習溜まり問題を避けながら価値を出せるか？
```

### 仮説D: 状況別フレーズが初期価値を強める

ポーカー卓、フロア、レストラン、移動など、使う場面はかなり決まっている。

検証したい問い:

```
全件ライブラリに加えて、状況別フォルダ/タグ/初期フレーズ集があると、実使用率が上がるか？
```

## 3. 見せるべきコンセプト案

プロトタイプ改修前でも、画面イメージまたは口頭説明で以下の4画面を見せれば十分。

### 画面1: 即翻訳

目的:

- 会話補助が入口になるか確認する
- 中→日需要を確認する

見せる内容:

- 日本語→中国語
- 中国語→日本語
- 音声入力
- 音声再生
- 「保存する」ボタン

確認質問:

```
この画面は、Google翻訳の代わりに会話中に開きそうですか？
開かないとしたら、理由はスピード・慣れ・精度・気まずさのどれですか？
```

### 画面2: 全件ライブラリ

目的:

- 保存されたものを全部見返せる価値を確認する

見せる内容:

- 保存した単語/フレーズ一覧
- 検索
- 最近使った順
- 状況タグ
- ドリル対象かどうかの表示

確認質問:

```
保存したものが全部ここに残るとしたら、見返しそうですか？
見返すなら、検索・日付・状況別のどれで探すことが多そうですか？
```

### 画面3: 状況別フォルダ

目的:

- 現地ポーカーFBの「任意フォルダ分け」ニーズを検証する

見せる内容:

- ポーカー卓
- フロア/手続き
- レストラン
- 移動
- ホテル
- 買い物
- 仕事
- 病院
- 子ども関連
- 自分で追加

確認質問:

```
自分なら、どんなフォルダを作りそうですか？
自動で分類される方がいいですか？それとも自分で任意に分けたいですか？
```

### 画面4: ドリル対象選択

目的:

- 全件ドリル化ではなく、覚えたいものだけ選ぶ設計を検証する

見せる内容:

- 「ドリルに追加」
- 「あとで見る」
- 「使用済」
- 「覚えた」
- 今日のドリル

確認質問:

```
翻訳したものが全部ドリルに入るのと、覚えたいものだけ選ぶのだと、どちらが続きそうですか？
```

## 4. 最初に見せる相手

優先順位は以下。

### 第1優先: 現地ポーカー・中〜高動機プレイヤー

理由:

- 既にプロトFBがある
- 現地で実使用できる
- 状況別フォルダの価値が検証しやすい
- 使用済ステータスが意味を持つ

見たい反応:

- 会話中/卓上で使うか
- Google翻訳ではなく使う理由があるか
- ポーカー卓/レストラン/フロア分類が自然か
- ドリル対象選択が面倒でないか

### 第2優先: 駐在帯同・高学習意欲の生活者

理由:

- ドリル化に強く反応
- 中→日と想定返答の需要が強い
- 生活フォルダ分類の価値が見える

見たい反応:

- 翻訳履歴をドリル化したいか
- 相手の返答想定が欲しいか
- 画像翻訳なしでも価値が成立するか

### 第3優先: 実務中国語・高習熟者

理由:

- 将来の高単価/ビジネス展開の品質基準になる

見たい反応:

- 複数候補文が必要か
- 機密情報保存がネックか
- 現MVPの延長で足りるか、別プロダクト級か

## 5. 成功/失敗基準

### 強い成功

以下が複数出る。

- 「これならGoogle翻訳より使う場面がある」
- 「保存されるならこっちを使う」
- 「このフォルダ分けは欲しい」
- 「この場面で使いたい」と具体場面が出る
- 「試したい」「今使いたい」
- 「いくらなら払う」または価格への具体反応
- 他の人に紹介できる

### 中間

- 便利そうだが、Google翻訳で足りる
- ドリルは良いが会話中には使わない
- 保存は良いがフォルダ分けまでは不要
- 無料なら使うが有料は不明

### 失敗

- Google翻訳/ChatGPTで十分
- 保存されるのが嫌
- ドリル化が重い
- 会話中に専用アプリを開く理由がない
- 状況別分類が面倒

## 6. 実装優先度（検証目的ベース）

### 今すぐ検証したい

| 優先 | 機能 | 理由 |
|---|---|---|
| 1 | 中国語→日本語入力/翻訳 | 複数ソースで最も強い未充足 |
| 2 | 全件ライブラリ | Google翻訳との差別化の土台 |
| 3 | ドリル対象の手動選択 | 006の反証に対応。全部ドリル化を避ける |
| 4 | 状況別フォルダ/タグ | 現地FBあり。探索性と実戦感を上げる |
| 5 | 発音再生/ピンイン/声調の安定 | 003/007/FBで価値が強い |

### 次に検証したい

| 優先 | 機能 | 理由 |
|---|---|---|
| 6 | 使用済ステータス | 現地利用者向けの差別化 |
| 7 | 類似/関連フレーズ | 007/FBで価値あり |
| 8 | 想定返答 | 聞き取り課題に接続できる |
| 9 | 属性別初期フレーズ | ポーカー/駐在/生活者で価値あり |
| 10 | 通知/予定化 | 継続率課題に対応 |

### 保留

| 機能 | 保留理由 |
|---|---|
| 画像翻訳 | 生活者には価値あり。ただしコア拡散リスク |
| ビジネス長文/候補文 | 008には強いが、初期MVPの主戦場ではない |
| 広東語対応 | 現時点では普通話優先 |
| 他ユーザーの共有フレーズ公開 | プライバシー/品質管理が必要 |
