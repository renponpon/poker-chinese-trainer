# Roadmap

最終更新: 2026-06-08

この文書は、既存の `docs/roadmap-2026-2028.md` を現在方針に合わせて短く再整理した実行用ロードマップです。中長期の詳細は元資料を参照します。

## 現在フェーズ

現在は、マカオ実地検証を中心に、Phrabit の中心価値が実利用で成立するかを確認するフェーズです。

優先すること:

- 実地ログを残す
- 保存・ドリル・再訪の行動を見る
- 明確なバグや小さな摩擦を直す
- Provider や音声入力の実用性を観察する

優先しないこと:

- 大規模リファクタ
- スコープ外の新機能
- ターゲットが曖昧な多機能化
- 検証前の課金機能作り込み

根拠:

- `AGENTS.md`
- `docs/phase2-runbook.md`
- `docs/macao-field-test-log.md`
- `docs/product-iteration-priority.md`

## 直近30日

目的:

保存・ドリル・再訪が行動として成立するかを見る。

やること:

1. マカオ/ポーカー現場での利用ログを継続する
2. 翻訳後に保存したか、ドリル追加したかを観察する
3. D1/D7 の再訪を確認する
4. 使われなかった場面を記録する
5. 明確なUI摩擦とバグだけ小さく直す

判断ポイント:

- 既存翻訳より Phrabit を開く理由が語られるか
- 保存したいフレーズが自発的に選ばれるか
- ドリル対象の選択が自然か
- 再訪が起きるか

### 仮の判定ライン

実地検証前の仮置きです。小サンプルでは率だけで判断せず、人数・具体発話・利用場面を合わせて見ます。

| 指標 | 最低ライン | 強いシグナル |
|------|------------|--------------|
| 観察数 | ポーカー現場または近接ユーザーで 5件以上 | 10件以上 |
| 既存翻訳ではなく開く理由 | 3件以上の具体発話 | 複数人が自発的に同じ理由を語る |
| 保存率 | 翻訳利用者の 30%以上、または 5人中2人以上が1件以上保存 | 翻訳利用者の 50%以上が保存 |
| ドリル追加率 | 保存者の 50%以上、または 5人中2人以上が1件以上ドリル追加 | 保存した人の大半がドリル追加まで進む |
| D1再訪 | 保存者の 30%以上、または 5人中2人以上が翌日以降に再訪 | D1再訪後に追加保存/追加ドリルが起きる |
| D7再訪 | 保存者の 15%以上、または 5人中1人以上が7日前後に再訪 | D7時点で「次に使うため」と言語化される |
| 課金価値 | 価格を聞く/支払い意思を語る人が1人以上 | 具体金額が複数人から出る |

このラインを下回った場合も即失敗とはしない。使われなかった場面、保存されなかった理由、既存解で十分だった理由を `docs/open-questions.md` に戻して、次の検証設計を修正する。

## Phase 2: 顧客実証

目的:

初期ビーチヘッドである海外ライブポーカー日本人に、Phrabit の課題と価値が成立するかを確認する。

主要タスク:

- ポーカー層の追加観察
- Good signal / Bad signal の整理
- 保存・ドリル・再訪ログの確認
- X募集や知人紹介による利用者追加
- ご意見番への軽いヒアリング

成功条件の候補:

- 直近の具体的な痛みが複数取れる
- 保存・ドリル追加が自発的に起きる
- D1/D7 の再訪が一部でも起きる
- 「次に自分で言いたい」という言葉が出る

根拠:

- `docs/customer-discovery.md`
- `docs/beachhead-outreach-templates.md`
- `docs/x-recruitment-draft.md`
- `docs/opinion-leader-candidates.md`

## Phase 3: 課金価値検証

目的:

Phrabit の価値が、無料利用を超えて支払い意思につながるかを確認する。

主要タスク:

- 支払い意思のインタビュー
- 価格帯の仮説作成
- 無料/登録不要からログイン・課金へつなぐ導線の検討
- 高精度/仕事モードの価値確認

確認する問い:

- 何に対して払うのか
- 月額か買い切りか
- 個人課金か会社経費か
- ポーカー層と業務層で価格許容が違うか

根拠:

- `docs/roadmap-2026-2028.md`
- `docs/open-questions.md`
- `docs/translation-provider-evaluation.md`

## Phase 4: セグメント拡張

目的:

ポーカー以外のどのセグメントに広げるべきかを判断する。

候補:

- 中国出張ビジネス層
- 駐在員
- 駐在帯同家族
- 高モチベーション学習者

主要タスク:

- CrowdWorks で非ビーチヘッド層を 3から5人ずつ深掘り
- セグメント別の課題文を比較
- 支払い意思と利用頻度を比較
- 業務利用時の保存・機密・音声入力の許容を確認

根拠:

- `docs/crowdworks-interview-recruitment.md`
- `docs/user-segments.md`
- `docs/problem-statements.md`

## Phase 5: 収益拡大

目的:

課金価値が確認できたセグメントで、継続利用と収益を伸ばす。

候補施策:

- 高精度/仕事モード
- 専門用語辞書
- STT強化
- 言語ペア拡張
- チーム/法人利用

注意:

このフェーズは、保存・ドリル・再訪と支払い意思が確認できた後に進めます。現時点では先に作り込まないほうが安全です。

根拠:

- `docs/roadmap-2026-2028.md`
- `docs/translation-provider-evaluation.md`

## 技術ロードマップ

| 領域 | 短期 | 中期 |
|------|------|------|
| 翻訳 | 現行の Azure / DeepL / Gemini 使い分けを維持 | 専門語・仕事モードの精度改善 |

| 利用計測 | `ai_usage_events` を活用 | 保存・再訪・ドリルの分析を強化 |
