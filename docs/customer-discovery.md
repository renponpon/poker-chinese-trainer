# 1. 製品仮説

製品仮説は、製品とその開発に関する創業チームの初期想定をまとめたもの。

## 1-1. 製品機能（主要10機能）

| # | 機能 | 概要 |
|---|---|---|
| 1 | 言いたかった日本語の入力 | テキストまたは音声で、その日言えなかった一言を即座に記録 |
| 2 | AI中国語生成 | Geminiが、教科書中国語ではなく現場で使う自然な中国語に変換 |
| 3 | ピンイン・解説の自動付与 | 声調付きピンインと、6見出しの実践的解説を同時生成 |
| 4 | 中国語音声再生 | Web Speech APIで生成フレーズを発音再生 |
| 5 | 自分専用フレーズライブラリ | 「自分用コード」で識別し、ローカル＋クラウドに保存 |
| 6 | 瞬間作文ドリル | 日本語→中国語の出題形式で反射定着までドリル化 |
| 7 | SRS（Anki SM-2ベース＋短期再学習） | Bad/Good/Perfectの3段階で次回復習日を自動調整 |
| 8 | 学習ステータス管理 | 新規/学習中/復習中/メンテ/習得の5段階で進捗可視化 |
| 9 | クラウド復元 | 別端末でも同じ自分用コードでフレーズと学習履歴ごと復元 |
| 10 | フィードバック導線 | Googleフォーム経由で改善要望を最短で送れる |

### コア機能（崩れたら製品が成立しない3つ）

- **AI中国語生成**
- **瞬間作文ドリル（SRS）**
- **自分のフレーズを入れられること**

残り7機能は支援機能扱い。

## 1-2. 製品のメリット

- 「現場の悔しさ」をその日のうちに学習素材に変えられる
- 翻訳アプリ・ChatGPTで終わらず、復習まで自動で繋がる
- 自分専用フレーズ集が貯まり、教材を探す必要がない
- ポーカー卓・カジノ・現地生活のリアル文脈に最適化
- スマホ前提のUIで、数分で1フレーズ追加できる

## 1-3. 知的財産

- 特許なし、出願予定なし
- 主な差別資産は、プロンプト・SRSロジック・ペルソナ理解・現場知見
- 中長期では「ポーカー / 海外カジノ / 現地生活」の現場フレーズデータが資産

## 1-4. 依存性分析

- Google Gemini APIの提供継続と価格安定
- Notion APIの提供継続
- Vercel低価格枠の継続
- スマホブラウザのWeb Speech API対応
- 中国語圏で日本人がプレイ・生活し続ける環境
- ChatGPT等が「保存＋SRS」まで一体化しないこと

## 1-5. 製品出荷スケジュール

| タイミング | やること |
|---|---|
| 〜1週間 | 知人3人テストの観察、Xで10人前後の公募開始 |
| 〜2週間 | 知人＋公募テスターの初期フィードバック収集 |
| 〜1ヶ月 | あなた自身がマカオで使う（ドッグフーディング）、現場で得た課題を即修正 |
| 〜2ヶ月 | 課金リリース。同時にポーカー以外の中国語圏ユーザーにもリーチ |

## 1-6. 所有・活用のための全体コスト（TCO・顧客側）

- 金銭: 月数百円〜千円台
- 時間: 1フレーズ追加1〜2分、1ドリル3〜10分
- 学習負荷: 教科書を新たに買う必要なし
- 切替コスト: ChatGPT等のメモを移す手間（実質ほぼなし）
- スマホ容量: ほぼゼロ（PWA）

# 2. 顧客仮説

## 2-1. センターピン痛み（カテゴリ非依存）

> 言いたかったのに言えなくて歯がゆい思いをした。  
> そのネガティブな気持ちを学習に向け、次の同じ場面で「言えた」という成功体験に変える。

状況ではなく感情の構造を捉えた軸。ポーカー卓・ミーティング・営業フロアなど、文脈が変わっても刺さる。

## 2-2. 顧客タイプ

| 段階 | 対象 | 役割 |
|---|---|---|
| ビーチヘッド | マカオ・海外ポーカーする日本人 | 課金検証層、物語の起点 |
| 第1拡張 | 中国・台湾出張ビジネス層（商社・コンサル・経営者・投資家） | 主課金層、地続きで発信できる |
| 第2拡張 | 駐在員 | 課金層、痛みは強いがリーチが課題 |
| 拡散役 | 中国語留学生 | 無料層、SNS拡散、口コミ |
| 保留 | 個人事業主の接客業（ホスト・ホステス・通訳・観光ガイド） | 後段で検討 |

### 留学生の扱いの議論

- カリキュラム・学校・先生がいるので、自分専用ドリルの必要性が低い
- 支払い意欲が低い
- 中国人と話す機会が多く「言えなかった」が起きにくい
- → 課金対象としては筋が悪く、無料層・拡散役として扱う

### 接客業を保留にした理由

- リーチ経路が不明（個人ではなく企業研修ルートが多い）
- 中国人富裕層相手の本気層は数が読めない
- 個人課金が起こる層（フリーランス通訳、ホスト・ホステス等）は別途検討

## 2-3. 顧客の課題

| 層 | 例 | 頻度 | 痛み強度 | 課金トリガー |
|---|---|---|---|---|
| A. 生活フリクション | タクシー、レストラン、ホテル | 毎日 | 3/10 | 弱（慣れる） |
| B. 社交フリクション | 卓上の雑談、笑いの輪 | 数日に1回 | 5/10 | 中（継続的） |
| C. クリティカル損失 | トラブル、誤解、金銭損失 | 月1〜2回 | 10/10 | 強（即購入） |

- **C層**: 財布を開かせる瞬間（高強度・低頻度）
- **A・B層**: 毎日アプリを開かせる継続力（低〜中強度・高頻度）

## 2-4. ROIの正当化

### ビーチヘッド層

- 1回のハプニングで $30〜$100 失う場面
- 月1〜2回は損失
- 仲間入りで情報・テーブル選択・連絡先で年数万円〜数十万円の機会差
- 月数千円なら明確に元が取れる

### ビジネス層

- 中国出張の成功確率に直結
- ChatGPTのバラバラ管理に困っている
- 経費精算可能なレンジ
- 月1480円〜2980円を即決できる経済力

## 2-5. 必要最小限の機能リスト

- 言いたかった日本語の入力
- AIによる自然な中国語生成（ピンイン・解説付き）
- 自分専用ライブラリ
- 瞬間作文ドリル（SRS）

これらは既に揃っている。今後の追加機能はビジネス面（オンボーディング、利用制限、決済）。

| 3 | ポーカーYouTuber・コーチへのギフトアクセス | 信頼ある発信源からの紹介 |

### 課金リリース以降

| 優先 | チャネル | 狙い |

| 4 | 中国語学習系note・YouTuberとの相互紹介 | 留学生・学習者全般を拡散役として活用 |

| プラン | 価格 | 内容 |
|---|---|---|
| Free | ¥0 | 月30フレーズまで生成、ドリル無制限 |

- LTV: 約8,880円
- CAC上限: 3,000〜4,000円程度（X運用・口コミ中心なら十分採算）

## 4-4. ChatGPT対抗のコアメッセージ

ChatGPTは「調べる」までは最強。  
でも、調べたフレーズを「覚える」「使う」までは持っていけない。  
仕組みがないから。  
  
SayItNext は、調べたフレーズを反射まで持っていくドリルとSRSが入っている。

### 3パターンのコピー使い分け

| 場面 | コピー |
|---|---|
| 短文（X用） | ChatGPTで調べた100フレーズ、何個覚えてますか？ |
| 中文（LP上） | ChatGPTで足りるのは「調べる」まで。次に同じ場面で口から出るかは別問題です。 |
| 長文（note用） | なぜChatGPTで中国語を調べても、現場で言えるようにならないのか。それは「調べる」と「覚える」「使う」の間に、誰も橋をかけていないからです。 |

## 4-5. 需要開拓ストーリー設計

- 「マカオのポーカーテーブルで言いたかったことが言えなかった話」

- 「中国人に話しかけられて何も返せなかった夜の話」

SEO・検索流入向け。中長期で効く。

# 5. 市場タイプ仮説

## 5-1. 市場タイプの判定

**既存市場の再セグメント化（ニッチ）**

| 判定根拠 | 内容 |
|---|---|
| 既存市場の存在 | 中国語学習アプリ市場（Duolingo, HelloChinese, SuperChinese 等） |
| 満たされていないニッチ | 自分の現場で言えなかった一言を起点にした、AI生成+SRS+自分専用ライブラリ |
| 既存リーダーが立てない理由 | 教材コンテンツ型ビジネスモデル、AI生成は自社カリキュラムと競合 |
| 新規市場ではない理由 | 「中国語を学びたい」需要は既に存在、啓蒙不要 |

## 5-2. リセグメント化市場チェックリスト

| 質問 | 回答 |
|---|---|
| 顧客はどの既存市場から移行してくるのか？ | Duolingo / HelloChinese / Anki / ChatGPT |
| それらの顧客の特徴は何か？ | 一定期間学習したが続かなかった / 現場で使えなかった層 |
| 既存サプライヤーが満たしていない重要なニーズは？ | 自分の文脈での反射定着、感情を学習に変える仕組み |
| 切り替えをうながす重要機能は？ | AI生成 + SRS + 自分専用ライブラリ |
| なぜ既存企業は同じ機能を提供できないのか？ | 教材コンテンツ型ビジネスモデルが障害 |
| 市場成長期間の見通し | 数ヶ月〜1年で初期PMF、その後の拡張に2〜3年 |
| 1〜3年の現実的な売上予測 | 1年目: 数百〜数千万円 / 3年目: 数千万〜1億円規模 |

## 5-3. ポジショニング軸

```
                  反射定着できる
                       │
                       │
  Anki, SuperChinese ──┼── SayItNext (●)
                       │
                       │
  教材中心 ────────────┼──────────── ユーザー固有
                       │
                       │
  Duolingo, HelloCN ──┼── ChatGPT, DeepL
                       │
                       │
                  単発理解で終わる
```

- 右上（**ユーザー固有 × 反射定着できる**）が空白
- ここが SayItNext のポジション
- 既存リーダーが構造的に立てない場所

## 5-4. ニッチが小さすぎる懸念への回答

- まず**中国語のニッチ**で既存ユーザーを奪う
- そこでPMFを作ったら、**他言語のリセグメント化**で同じパターンを横展開
- 各言語ごとに「教材中心の既存リーダー」がいるので、毎回同じ攻め方が通用する

つまりリセグメント化は **1言語完結ではなく、複数言語で繰り返せる戦略**。新規市場創造より勝率が高い。

## A-3. 紹介のための3層構造

```
あなた → ポーカー仲間（紹介者） → 紹介先（インタビュー対象）
        ↑                       ↑
        Layer A                Layer B/C
```

| Layer | 用途 | 設計ポイント |
|---|---|---|
| **A: 仲間向けDM** | 紹介を頼む | 「1人だけでOK」「コピペで転送するだけ」 |
| **B: 転送用紹介文** | 仲間が紹介先に送る | 信頼性・時間・売り込みでない明示・Calendly |
| **C: ファーストDM** | 紹介先からの返信に返す | 目的を再明示・予約リンクで摩擦ゼロ |

詳細テンプレ: [`beachhead-outreach-templates.md`](./beachhead-outreach-templates.md)（LINE / X・カテゴリ別）

| **プロトタイプ試用** | 実際に使ってもらい反応を見る | 重要 | コアな興味を持つ人だけ |

| セグメント | 今やること | プロトタイプを見せる？ |
|---|---|---|
| ビーチヘッド（ポーカー30人） | 試用＋インタビュー | YES（既に進行中） |

1. 「プロトタイプ」ではなく「方向性」を見せる
2. イノベーターは完成度より「早く触れること」に価値を感じる
3. 待つほど学習機会の損失コストが大きい

# B. 課題プレゼンテーション

**目的**: 課題仮説を顧客にぶつけて、反応で検証する。**製品は出さない**。

## B-1. 検証する課題仮説ステートメント（1文）

> 「中国語圏で日常的に中国語に触れる日本人は、現場で『言いたい』と思った言葉が、既存ツール（Duolingo / ChatGPT / 翻訳アプリ / 語学学校）では『反射的に出る』レベルまで定着しないため、人間関係・社交・金銭・自尊心の損失を繰り返している。」

## B-2. ステートメントを構成する4要素

| # | 要素 | 検証ポイント |
|---|---|---|
| 1 | 対象: 中国語圏で日常的に中国語に触れる日本人 | この属性で本当に痛みがあるか |
| 2 | トリガー: 現場で「言いたい」と思う瞬間 | この瞬間が頻繁に起きるか |
| 3 | 失敗構造: 既存ツールでは反射定着しない | 既存ツールが本当に効いていないか |
| 4 | 損失: 人間関係・社交・金銭・自尊心 | 損失が「痛い」レベルか |

インタビューで「4つすべて」が確認できれば仮説は強い。3つ以下なら絞り込みが必要。

| 3 | 今、人々がやっていること | Duolingo / ChatGPT / 学校 / メモ / 諦め — それぞれの失敗 |
| 4 | 既存解の構造的欠陥 | 「調べる」と「使える」の間の橋が無い／個人化・反射定着・継続性 |
| 5 | あなたに聞きたいこと | 一致するか、どこが一番痛いか、現在の対処、解決後の理想 |

## B-4. 実施フォーマット

- 「説明」ではなく「呼び水」として使う

## B-5. 観察すべき「痛みワード」と「シグナル」

### 🟢 強い成功シグナル

- 「そう、それ！」「まさにそれ」
- 「最近もあって…」（直近エピソード自発）
- 数字（◯円損した、◯回経験）
- 感情ワード（悔しい / 恥ずかしい / ナメられた）
- 既存ツールへの具体的不満
- 「いくらなら払う」「人に紹介する」を自発

### 🟡 中間シグナル

- 「たまにありますね」
- 「自分は大丈夫だが周りは」
- 痛みあるが致命的でない

### 🔴 失敗シグナル

- 「困ってない」
- 「ChatGPTで十分」
- 反応が薄い、エピソードが具体的に出ない
- 質問が製品スペックに向く

## B-6. 全体の成功 / 失敗判定（5〜10件後）

| 状態 | 基準 |
|---|---|
| 仮説が **強い** | 対象セグメント50%以上で強い成功シグナル、3つ以上のセグメントで痛みが共通 |
| 仮説が **弱い** | 「困ってない」「ChatGPTで十分」が多数派、痛みが抽象的 |
| 仮説の **再構築が必要** | セグメントによって反応が大きく異なる、痛みが中国語より根本的な何か |

## B-7. 関連ドキュメント

- **`docs/interview-playbook.md`**: インタビュー実施用プレイブック（A4両面）
- **`docs/interview-observation-sheet.md`**: インタビュー後3分以内の観察記録シート（1人1枚）

# C. 顧客についての理解の深化

**目的**: パーソナとセグメントの解像度を、5〜10件のインタビューで上げる。

## C-1. ペルソナ精緻化のフレーム（インタビューから埋める）

各セグメント（ポーカー / ビジネス / 駐在員 / 留学）について、以下を埋める。

| 項目 | 内容 |
|---|---|
| 1日のルーティン | 朝〜夜の中国語接触ポイント |
| 中国語との接触頻度 | 週何回、どんな場面 |
| 既に試した解決策と挫折ポイント | 何を、いつ、なぜやめた |
| 痛みの瞬間トップ3 | 直近半年でランキング |
| 解決後の理想像 | 1年後どう変わっていたい |
| 情報収集チャネル | X / note / YouTube / 友人 |
| 影響力（紹介力） | 友人何人に紹介できる立場か |
| 支払い意思 | 月額いくらまでなら |

## C-2. 横断分析の集計タグ

5〜10件溜まったら、観察シートのタグで集計する。

- 痛みのタイプ: 金銭 / 関係性 / 自尊心 / 機会 / 体験品質
- 頻度クラス: 高頻度 / 中頻度 / 低頻度
- 支払い意思: 強い / 中 / 弱い
- 伝道力: 強い / 中 / 弱い
- イノベーター度: 高 / 中 / 低

## C-3. アーリーアダプター（イノベーター）の特徴抽出

5〜10件の中で「最も熱量が高い人」の共通点を見つけ、**アーリーアダプター・ペルソナ**を再定義する。

候補となる特徴（仮説）:

- 中国語学習を**自己流で組んでいる**（ChatGPTやAnkiを既に使っている）
- 新しいアプリを**1ヶ月で3つ以上試している**
- **直近半年で**中国語に困った具体エピソードを2つ以上持つ
- 自分の課題を**自分で言語化できる**
- SNSで**何かを発信している**（伝道力）

→ 検証中にこの仮説を更新する。

## C-4. セグメント別の優先度判定

5〜10件の収集後、以下の3軸でセグメントを順位付けする。

| 軸 | 指標 |
|---|---|
| **痛みの強さ** | 平均痛みスコア（観察シートより） |
| **市場規模** | 推定人口 × 接触可能性 |
| **支払い意思** | 自発的言及の比率 |

3軸合計で**ビーチヘッド再定義**。当初はポーカー想定だが、結果次第で第1拡張に重心を移す可能性。

# D. 市場知識の蓄積

**目的**: 業界・競合・代替手段の構造を理解し、ポジショニングを精緻化する。

## D-1. 競合・代替手段マップ（最新）

| カテゴリ | 代表 | 強み | 弱み | 当方の差別化 |
|---|---|---|---|---|
| 大手語学アプリ | Duolingo / Babbel | 認知度・コンテンツ量 | 個人化なし・現場とズレ | 個人化 × 反射定着 |
| 中国語特化アプリ | SuperChinese / HelloChinese | 中国語に最適化 | 教科書的、現場と乖離 | 現場フレーズ生成 |
| AI翻訳 | ChatGPT / DeepL | 即時翻訳 | 「調べる」で終わる | 「覚える・使う」まで |
| 学習サービス | italki / Cafetalk | 人間との対話 | 高コスト・継続困難 | 低コスト・自分のペース |
| メモアプリ | Anki | カスタマイズ性 | 設計が高度、続かない | デフォルトで現場最適 |

## D-2. 業界専門家からの情報収集

ご意見番カテゴリの3〜5名から、以下を聞き出す。

- 中国語学習市場の規模感（学習者数、課金規模）
- 既存サービスの裏側（どのアプリが伸びていて、なぜ）
- 学習者の典型的な離脱パターン
- 「現場」と「学習」の橋を架けようとした既存の試み
