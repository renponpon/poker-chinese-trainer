## 判定基準の候補

| 検証対象 | 見たいシグナル |
|----------|----------------|
| 初回価値 | 既存翻訳ではなく Phrabit を開いた理由が語られる |
| 保存価値 | 翻訳後に保存したいフレーズが自発的に選ばれる |
| ドリル価値 | 保存したフレーズを後で練習したいと言う、または実際に戻る |
| セグメント適合 | 直近エピソード、痛み、既存解の不満、支払い意思が揃う |
| 課金価値 | 「いくらなら払う」が自発的、または具体的に回答される |

## 最初の判定ライン

現在の仮置きは `docs/roadmap.md` の「直近30日」に置く。主な最低ラインは以下。

| 指標 | 仮の最低ライン |
|------|----------------|
| 観察数 | ポーカー現場または近接ユーザーで 5件以上 |
| 開く理由 | 既存翻訳ではなく Phrabit を開く理由が3件以上語られる |
| 保存 | 翻訳利用者の 30%以上、または 5人中2人以上が1件以上保存 |
| ドリル追加 | 保存者の 50%以上、または 5人中2人以上が1件以上ドリル追加 |
| D1再訪 | 保存者の 30%以上、または 5人中2人以上が翌日以降に再訪 |
| D7再訪 | 保存者の 15%以上、または 5人中1人以上が7日前後に再訪 |
| 課金価値 | 価格を聞く/支払い意思を語る人が1人以上 |

この数値は検証を締めるための初期ラインであり、プロダクトの合否を単独で決めるものではない。発話、利用場面、使われなかった理由とセットで判断する。

根拠:

- `docs/interview-playbook.md`
- `docs/interview-observation-sheet.md`
- `docs/product-concept-validation-plan.md`
