  - 現状は「フレーズの日本語→中国語化」だけ。

- **SRSステータスに「使用済」を追加**（新規→学習中→完了→**使用済**）。
  - 完了後に実際に現地で使ったら**手動で「使用済」に変更**するイメージ。
  - 海外滞在者向けアプリならではの体験設計。

### 🅲 ユーザー自身のアイデア

- **即翻訳が会話中にも使えれば、仕事中にもアプリ活用可能**。その場の活用内容を自動でメモ送りすれば、**100%実践的な内容のドリル**が作れるのでは？
  - 検討余地: 翻訳スピード、アプリを開く手間の時短方法。
- **プライベート内容を100%排除できれば、他ユーザーの参考フレーズで質の良いドリル**を作れそう。
- **毎日利用してもらう仕掛けが鍵。** Duolingo は超頻繁な通知＋連続学習の可視化・価値化でモチベ維持している。
- **対象局面でアプリを開いてメモしてもらう必要**がある。
  - 多くは相手との**会話の時**だから、**中国語→日本語翻訳機能**を付けて、会話時にこのアプリを開きたくなる作りに。
- 個人で「知りたいフレーズ」を思いついた時にツール入力するより、**会話中にツール入力する場面の方が多そう**。
  - その場合、**中→日変換**があった方が会話が円滑になり、利用頻度が上がる可能性。

## 🧩 浮かんでいる論点・未決事項（一段落後の検討用）

> 着手前に必ず議論したい争点を一覧化。インタビュー結果と合わせて判断する。

### 論点1: **コアバリューの方向性 ─「学習」と「会話」のどちらに寄せるか**

- 🅰の「勉強と会話の両方で使いたい、片方が片方のモチベを上げる」
- 🅲の「会話中にツール入力する場面の方が多そう」「会話時にアプリを開きたくなる作りに」
- → **会話補助（中→日含む翻訳機能）をコアに据え、学習（ドリル）がその副産物として走る**設計が、利用頻度・継続率の両方で勝ち筋になり得る。
- 一方、**観察シート003（あおちゃん）が言語化したコア仮説**「覚える→反射で出る」は**学習側**。この層を捨てるのか、両立できるのか要判断。

### 論点2: **インプット方法を増やすか／コアを絞るか**

- 🅱「インプット方法を増やすと使用機会が増える」（画像認識・数字・単語・任意の疑問）
- ユーザー自身も認識: **コアがブレるリスク** ＆ **インプット量が膨大化** すると未学習が溜まる
- 既存検討中の機能（他ユーザーのフレーズ参照／AIコーチとの対話中フレーズもドリルに追加）と合わせると、**未学習の山がユーザーモチベを下げるリスク**。
- → **「全部解決」型 vs 「特化ツール」型**の方針決定が必要。一段落後の最重要論点候補。

### 論点3: **継続率の設計（予定化と通知）**

- 観察シート005（みとちゃん）の本人発の自己分析：「アプリ・YouTubeは予定に入ってないから続かない／レッスン系は予定に入ってるから続く」
- 🅲「Duolingoは超頻繁な通知＋連続学習の可視化・価値化でモチベ維持」
- → **通知・連続記録・予定化要素**は継続率施策の必須項目候補。**対人要素（AIコーチ／人とのレッスン枠予約）**まで踏み込むかも要判断。

### 論点4: **中国語→日本語方向の実装優先度**

- 観察シート003（あおちゃん）追加要望
- 観察シート005（みとちゃん）の「中国人同士の会話が気になる」疎外感
- 🅰の明示要望
- 🅲の「会話中の入力場面の方が多い」推測
- → **3つの独立ソースから一致**。**優先度は高い**と推測されるが、コアバリュー（学習）との接続設計が必要。

### 論点5: **「使用済」ステータスの実装可否**

- 🅱の提案
- 海外滞在者向け体験を強める差別化要素になり得る
- 実装コスト低（SRSにステータス1つ追加）／心理的価値高（達成感の可視化）
- → 一段落後にプロトタイプ実装テスト候補。

### 論点6: **未学習溜まり問題への対処**

- インプット拡張すると確実に発生。
- 🅳の「全件ライブラリ＋任意の状況別フォルダ」要望は、未学習が増えた後の探索性・整理性を上げる方向の候補。
- 候補となる対処：
  - 自動間引き（古い未学習を「保留」に逃がす）
  - 重要度フラグ（卓上で実際に使うかどうかでタグ）
  - 個別ユーザーのスループットに応じた追加制限
- → 機能拡張前に**未学習溜まりの体験計測**をする方が安全。

### 論点7: **会話補助に寄せるなら、翻訳スピードとアプリ起動の手間がボトルネック**

- 🅲「翻訳スピード、アプリを開く手間の時短方法」
- ホーム画面アイコン、ウィジェット、ショートカット、共有シート連携など実装上の選択肢を要検討。
